# Review — 复习模块

> **完整规格**：[DEV_SPEC.md 第 8.3 节](../../../DEV_SPEC.md#83-复习模块-review)
> **实现优先级**：P0-P1

## 触发条件

用户消息包含以下关键词之一：复习、review、考我、提问、quiz、test、抽题、来几题

## CWD 边界

NEVER access files outside CWD。所有操作在 `{CWD}/InterviewVault/` 内完成。

## 前置检查

1. 检查 `InterviewVault/.progress/schedule.json` 是否存在且有条目。
   - 若无条目 → 提示用户先使用 `learn` 学习内容。
2. 读取 `config.yaml` 获取：
   - `review.default_mode`（FAST | DEEP | INTERVIEW）
   - `review.questions_per_session`（默认 10）

## 模式选择

根据用户输入和配置确定交互模式：

| 用户输入关键词 | 模式 |
|---------------|------|
| "深度复习" / "deep" / "详细点评" | DEEP |
| "模拟面试" / "interview" / "面试模式" | INTERVIEW |
| 无特殊指定 | `config.yaml` 中的 `default_mode` |

**模式切换**：用户可在每轮复习前说"切换到 DEEP 模式"临时切换。

## 主流程 — FAST 模式（P0 已实现）

详见下方 "FAST 模式交互" 节。

**特点**：仅展示问题 + 用户回答 + AI 简单评分（1-5）+ 简短补充。
**追问**：0 轮。

## 主流程 — DEEP 模式（P1）

### 步骤 1-2：与 FAST 模式相同

拉取待复习题目、展示问题、收集回答。

### 步骤 3：AI 详细评估

对比用户回答与参考答案，输出结构化点评：

```
✅ 评分：{score}/5 — {评语}

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 详细点评
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 你说对了：
  • {用户回答中正确的要点 1}
  • {用户回答中正确的要点 2}
  • ...

⚠️ 需要补充：
  • {用户遗漏的踩分点 1}
  • {用户遗漏的踩分点 2}
  • ...

📖 完整参考答案：
  {结构化答案，分点列出，踩分点加 **bold**}

💡 记忆口诀 / 模式总结：
  {该题涉及的核心模式或口诀}
```

**评分标准**（同 FAST）：

| 分数 | 含义 | 判断标准 |
|------|------|---------|
| 5 | 完美 | 完全正确，要点齐全 |
| 4 | 正确但犹豫 | 正确但略有遗漏或表述不够精准 |
| 3 | 部分正确 | 部分要点正确，有明显遗漏 |
| 2 | 错误 | 关键概念错误 |
| 1 | 完全不记得 | 完全不会 |

### 步骤 4：单轮追问（可选）

**触发条件**：评分 >= 3 且用户回答有遗漏点时触发。

追问内容来源：
1. 参考答案中的踩分点（用户未提及的点）
2. 与当前概念相关的其他知识点
3. 设计取舍：为什么这样设计，替代方案对比

**追问格式**：

```
─── 追问 ───

💡 {简短提示语，引导用户思考遗漏点}

追问：{基于遗漏点生成的问题}

💭 请补充回答：
```

**追问评分**：对追问回答给出 1-3 分，合并到原题评分中：
- 追问回答优秀（3 分）：原题评分 +0（已达上限）
- 追问回答正确（2 分）：原题评分不变
- 追问回答错误或不会（1 分）：原题评分 -1（最低降到 3）

用户可随时说"跳过"或"不会"结束追问。

### 步骤 5-7：与 FAST 模式相同

更新 SM-2 调度、记录错误笔记（score <= 2）、生成会话记录、更新统计。

**会话记录 mode 字段**：`"DEEP"`

## 主流程 — INTERVIEW 模式（P1）

### 步骤 1-2：与 FAST 模式相同

### 步骤 3：面试场景化提问

以面试官视角提出问题：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎤 模拟面试 — 第 {current}/{total} 题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

面试官：请你谈谈 {问题}？

💭 请回答：
```

### 步骤 4：多轮追问（最多 3 轮）

**追问触发条件**：
- 用户回答正确但不够完整 → 追问遗漏点
- 用户回答有错误 → 先纠正错误，再问相关延伸
- 用户回答优秀 → 提升难度，问更深的问题
- 用户说"不会" → 直接给答案，不追问

**追问内容来源**：
1. 参考答案中的踩分点（用户未提及的点）
2. 常见错误（用户可能踩的坑）
3. 关联概念（与当前概念相关的其他知识点）
4. 设计取舍（为什么这样设计，替代方案对比）

**追问格式**：

```
─── 第 {round} 轮追问 ───

✅ 答得好：{肯定用户回答中的正确部分}

💡 提示：{引导提示}

追问：{具体问题}

💭 请回答：
```

**追问规则**：
- 每轮追问最多 3 轮
- 用户可随时说"跳过"或"不会"结束追问
- 薄弱概念（status == "weak"）的追问不得重复上一轮相同问题
  - 上轮问 "What" → 本轮问 "How" 或 "Why"
  - 上轮问理论 → 本轮问场景/调试/对比
  - 上轮问正向 → 本轮问边界条件/异常处理

### 步骤 5：多维度评分

调用 scoring.py 计算四维度评分：

```bash
python .claude/skills/review/scripts/scoring.py \
  '{"accuracy":{N},"depth":{N},"code_relevance":{N},"design_thinking":{N}}'
```

**评分维度**（各 0-10 分）：

| 维度 | 含义 | 高分标准 |
|------|------|---------|
| 准确性 | 事实正确性 | 概念无错误，定义准确 |
| 深度 | 是否触及原理层面 | 能解释 "为什么" 而非仅 "是什么" |
| 代码关联 | 是否结合代码/实现细节 | 提到具体类名、方法、JVM 参数 |
| 设计思维 | 对比/权衡/扩展思考 | 能比较不同方案，说明适用场景 |

**评分输出**：

```
📊 本题评分：

| 维度 | 分数 | 说明 |
|------|------|------|
| 准确性 | {N}/10 | {说明} |
| 深度 | {N}/10 | {说明} |
| 代码关联 | {N}/10 | {说明} |
| 设计思维 | {N}/10 | {说明} |

🏆 综合评分: {overall}/10  (等级: {grade})

📖 学习建议：
  {具体建议，如"建议阅读 01-Notes/... 中关于 XXX 的部分"}
```

**综合评分转换 SM-2 分数**：
- overall >= 9 → SM-2 score = 5
- overall >= 7 → SM-2 score = 4
- overall >= 5 → SM-2 score = 3
- overall >= 3 → SM-2 score = 2
- overall < 3 → SM-2 score = 1

### 步骤 6-8：与 FAST 模式相同

更新 SM-2 调度、记录错误笔记（SM-2 score <= 2）、生成会话记录、更新统计。

**会话记录 mode 字段**：`"INTERVIEW"`

## 通用流程（所有模式共享）

### 拉取待复习题目

1. Read `InterviewVault/.progress/schedule.json`
2. 筛选 `next_review <= today` 的条目
3. 排除 `status == "mastered"`（除非用户明确要求）
4. 排序规则：
   - 第一优先级：`frequency` = 必问 > 常问 > 冷门
   - 第二优先级：正确率低的优先
   - 第三优先级：过期天数多的优先
5. 取前 `questions_per_session` 道（默认 10，用户可临时指定）

### FAST 模式交互

#### 展示问题

```
--- Q{current}/{total} ---
📚 {domain} | {frequency}

❓ {问题原文}

💭 请回答（直接输入答案）：
```

#### 展示评分

```
✅ 评分：{score}/5 — {评语}

💡 补充：{简要补充或纠正（1-2 句话）}

[Enter 继续下一题]
```

### 更新 SM-2 调度

每道题评分后，调用 sm2.py 更新：

```bash
python .claude/skills/review/scripts/sm2.py \
  InterviewVault/.progress/schedule.json \
  {question_id} \
  {score}
```

将 sm2.py 输出的 JSON 写回 `schedule.json`（原子写入）。

SM-2 更新规则：

| 评分 | interval | ease_factor | consecutive_correct | status |
|------|----------|------------|-------------------|--------|
| 5 完美 | × ease_factor | 不变 | +1 | learning |
| 4 正确 | × ease_factor | 不变 | +1 | learning |
| 3 部分 | max(1, × 0.5) | × 0.8 | 0 | learning |
| 2 错误 | max(1, × 0.3) | × 0.7 | 0 | weak |
| 1 全忘 | =1 | × 0.5 | 0 | weak |

**mastered 判定**（P1 启用）：
- `consecutive_correct >= 5` 且 `ease_factor >= 2.5` → 自动标记 `status = "mastered"`
- `interval` 设为 365
- 不再主动推荐复习，但用户可手动触发

### 记录错误笔记（评分 <= 2 时）

若 SM-2 评分 <= 2：
1. Read 对应笔记文件
2. 在文件末尾追加错误笔记：

```markdown
### 错误笔记

**{YYYY-MM-DD} — {question_id}**
- **问题**：{问题原文}
- **错误回答**：{用户回答摘要}
- **混淆点**：{AI 分析的用户误解}
- **正确理解**：{正确知识点}
- **关联概念**：[[...]]
```

3. 同时检查 `03-Exam-Traps/{Domain}.md`：
   - 若该概念无陷阱条目，追加一个
   - 若已有，在对应条目下追加错误记录

### 生成会话记录

全部题目完成后：

1. 计算统计：
   - 平均得分
   - 掌握题数（FAST/DEEP: >= 4 分；INTERVIEW: >= 7 分）
   - 需加强题数

2. 生成会话文件：`04-Sessions/{YYYY-MM-DD}_review.md`

```markdown
---
date: "{YYYY-MM-DD}"
mode: "{FAST|DEEP|INTERVIEW}"
total_questions: {N}

# 复习会话 — {YYYY-MM-DD}

- 模式：{mode}
- 题目数：{N}

## 详细记录

### Q1: {question_id} — {概念}

- **得分**：{score}
- **用户回答摘要**：{摘要}
- **薄弱点**：{如有}
- **建议**：{如有}

### Q2: ...

## 更新计划

| 题目 | 下次复习 | 间隔 | ease_factor |
|------|---------|------|------------|
| {qid} | {date} | {interval} 天 | {ef} |
```

## 出题规则（Quiz Rules）

### 零提示原则

- 问题描述不能暗示正确答案
- 选项描述必须中性，不含 "(Recommended)" 等暗示
- 正确答案位置必须随机化，不能固定在第一或最后

### 薄弱概念追问不重样

- 同一薄弱概念（status == "weak"）的追问，不得重复上一轮的相同问题
- 必须换场景、换角度重新表述：
  - 上轮问 "What" → 本轮问 "How" 或 "Why"
  - 上轮问理论 → 本轮问场景/调试/对比
  - 上轮问正向 → 本轮问边界条件/异常处理

### 难度动态平衡

| 复习场景 | 易(⭐) | 中(⭐⭐) | 难(⭐⭐⭐) |
|----------|--------|----------|----------|
| 日常复习 | 40% | 40% | 20% |
| 薄弱区训练 | 0% | 30% | 70% |
| 考前冲刺 | 20% | 50% | 30% |

### 题型多样化

每轮至少覆盖 3 种题型：
1. **事实回忆**："HTTP 状态码 503 表示什么？"
2. **概念理解**："为什么 JVM 选择可达性分析而非引用计数？"
3. **行为预测**："如果 G1 的 Remembered Set 丢失，会发生什么？"
4. **比较区分**："CMS 和 G1 在碎片处理上有什么区别？"
5. **调试场景**："给定这个 GC 日志，最可能是什么问题？"

## 错误处理

| 场景 | 处理方式 |
|------|---------|
| 无待复习题目 | 提示"今日无复习任务，明天再来" |
| schedule.json 损坏 | 报错，建议从备份恢复 |
| 评分参数错误 | 拒绝非法评分（非 1-5） |
| 用户中途退出 | 已完成的题目仍更新，生成部分会话记录 |
| 追问时用户说"不会" | 直接给答案，结束追问 |
